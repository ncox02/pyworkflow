The core idea

Before the details, here's the whole picture.

A modern Python developer uses a handful of tools — maybe four or five — that each play a specific role. Once you understand what each one is for, everything else clicks into place. The tools change. The workflow doesn't.

The landscape feels overwhelming because there are hundreds of tools. But they all do one of five things. Once you see the categories, you can evaluate any new tool instantly.

The Editor / IDE

Where you write and read code. An IDE (Integrated Development Environment) is a text editor with superpowers — it understands your code, highlights errors, helps you navigate, and runs your programs.

A framework for where you are

What level are you at?

A useful way to think about AI coding tools — described by researcher Kyle Jensen — is a ladder with distinct rungs. Most people start at Level 0 or 1. Understanding where you are helps you know what's possible next.

Level 0–1

Browser chat + copy-paste

You're using ChatGPT or Claude in a browser tab. You describe a problem, get code back, paste it into your editor, run it, hit an error, copy the error back up, and repeat. Or you have Copilot in your IDE doing inline suggestions — helpful, but still basically a chat loop. This is where most people start. This is where most students are.

There's nothing wrong with this. It works. But it's slow, and there's a lot more available.
Level 2

Agentic IDE (Cursor)

You have an AI-first editor like Cursor. You tell the AI what you want, and it edits files, reads your codebase, and works across multiple files at once — still inside the editor, but with much more autonomy. The AI isn't just suggesting; it's doing. You review diffs and decide what to keep.

This is where the IDE deep-dive pages on this site are targeted.
Level 3

Dedicated coding agents (Claude Code)

Tools like Claude Code, OpenAI Codex CLI, and Gemini CLI run in your terminal and have access to your full computing environment — not just open editor files. They can think about a task, read and write any file, run scripts, see the output, and iterate autonomously. The IDE becomes optional; the terminal is the interface.

This is what the Claude Advanced section covers.
Level 4

Connected agents (MCP tools)

You add external capabilities to your agent via MCP (Model Context Protocol) — connecting it to databases, live documentation, web search, and other APIs. The agent can now pull current library docs, query your database, or search the web while it codes. Superpowers, Context7, and Sequential Thinking from the Claude Advanced page are Level 4 tools.

Level 5

Fully autonomous runs

You give the agent a goal, set it running in a container, and come back an hour later. This is what people mean when they say "AI wrote a whole feature by itself." It's real, but requires significant setup, careful scoping, and thorough review of the result. Not inherently better — just more autonomous.

All levels use the same underlying AI model. The difference is how much structure, access, and autonomy you give it.
"The tools change.
The workflow doesn't."

Whether you're using VS Code or PyCharm, Copilot or Claude, pip or poetry — the underlying pattern is identical. You write code in an editor, manage your dependencies in an environment, and save and share your work with Git.

Learning the pattern means you can switch tools at any time without starting from scratch. That's the goal of this site.
