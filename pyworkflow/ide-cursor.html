What Cursor actually is

Cursor is a code editor built by Anysphere and launched in 2023. It started as a fork of VS Code — meaning it took VS Code's source code and rebuilt on top of it. This means everything you know from VS Code works: your extensions, your keyboard shortcuts, your themes, your settings. You can import your entire VS Code configuration in one click.

But Cursor made a fundamentally different architectural choice: AI is not a plugin — it is the editor. Where VS Code adds AI via extensions that run alongside the editor, Cursor built AI into the core editing experience. The result is AI that can read your entire codebase simultaneously, understand context across multiple files, and take actions across your project — not just the file you have open.

💡
If you already use VS Code, switching to Cursor costs you nothing

On first launch, Cursor asks to import your VS Code settings, extensions, and keybindings. The result is an editor that feels completely familiar — but with a more powerful AI layer underneath. You can evaluate Cursor without abandoning anything.

Cursor's AI modes — what each one does

Cursor has four distinct ways to interact with AI. Understanding which to use when is the key to using Cursor well.

Tab

Inline autocomplete

The most passive mode. As you type, Cursor suggests what comes next in gray text. Press Tab to accept. Unlike Copilot, Cursor's completions are informed by your entire codebase — so they're more likely to match your patterns, your variable names, and your architecture.

Best for: Day-to-day typing. Writing functions that fit your existing code style. Repetitive patterns.
⌘K

Inline edit

Select code → press Cmd+K → describe what you want changed. The AI rewrites the selection in place and shows you a diff (before/after). You accept (Cmd+Y) or reject (Cmd+N). This is the fastest way to make targeted changes: "Add type hints," "Handle the null case," "Rewrite this as a list comprehension."

Select a function → ⌘K → type:
"add try/except to handle network errors, log them, and return None on failure"
Best for: Targeted rewrites, adding error handling, type hints, refactoring a specific block, improving clarity.
⌘L

Chat

A conversation panel where you can ask questions, get explanations, and discuss your code. The key feature: you can reference your codebase with @ symbols. @filename.py attaches that file's contents. @codebase lets the AI search your entire project semantically. @web lets it search the internet.

Reference syntax
  • @filename.py — attach a specific file
  • @codebase — AI searches your entire project
  • @web — AI searches the internet
  • @docs — AI searches indexed documentation
Best for: Understanding code, design questions, "how do I..." questions, getting explanations.
Agent

Agentic mode

The most powerful mode. Give the AI a goal — not a specific code change, but an outcome — and it figures out what to do. It reads files, writes code, creates files, runs terminal commands, reads the output, fixes errors, and iterates until the task is done. You watch and review each step.

What you can say
  • "Set up a Flask API with three endpoints: GET /data, POST /upload, DELETE /remove"
  • "Write tests for everything in utils.py and make sure they all pass"
  • "Find all functions with no type hints and add them"
  • "Refactor the database module to use SQLAlchemy instead of raw SQL"
⚠
Always review agent output

Agent mode can make changes across many files. Always read what it produced before accepting. It gets tasks right most of the time, but it can also confidently write plausible-looking code that has subtle bugs. Your judgment is still the last line of review.

Best for: Large features, multi-file refactors, repetitive changes across a codebase, generating scaffolding.

A real session in Cursor

Starting a new feature

Open chat → describe what you want to build

Press Cmd+L. Type: "I want to add a function that reads a CSV file, validates that all required columns are present, and returns a cleaned DataFrame. Use @codebase to see what column names I'm already using elsewhere." Cursor searches your project, finds the relevant patterns, and suggests an implementation consistent with your existing code.

Writing the code

Tab completions inform your typing

As you write, inline suggestions appear that match your existing patterns — your naming conventions, your error handling style, your import preferences. Accept what fits, type through what doesn't. The completions feel less generic than Copilot because they're based on your actual codebase.

Something isn't right

Select the problem → ⌘K → describe the fix

You see a function that's too long and confusing. Select it. Press Cmd+K. Type: "Split this into three smaller functions with clear names." The diff appears — the original is replaced by three focused functions. You review and accept.

Big task

Switch to agent → give a goal

You want tests for your entire module. Switch to Agent mode. Type: "Write comprehensive pytest tests for everything in analysis.py. Run them and fix any failures." Cursor reads the file, generates tests, runs them in the terminal, sees which ones fail, fixes them, and reports back. You review the final test file.

Reviewing changes

Read every diff before accepting

Cursor shows you diffs for every change it makes. Read them. This is not optional. Cursor is fast and often right, but it can also produce code that looks correct and has a subtle bug. Your review is the quality gate.

Choosing your model

One of Cursor's most distinctive features: you choose which AI model powers it. Different models have different strengths. You can switch per task.

Claude Sonnet / Opus
Anthropic

Cursor's default and most popular choice. Excellent at reasoning, following complex instructions, and understanding large codebases. Claude Sonnet balances quality and speed; Opus is slower but stronger for very complex tasks.

Best for: Complex reasoning, architecture questions, large refactors
GPT-4o
OpenAI

Fast and capable. Good general-purpose coding model. Slightly weaker than Claude at following very detailed instructions, but competitive for most tasks.

Best for: Fast chat answers, quick completions, general coding help
Gemini 1.5 Pro
Google

Extremely large context window (1M tokens) — can read more of your codebase simultaneously than any other model. Strong at tasks requiring comprehensive codebase understanding.

Best for: Very large codebases, tasks requiring reading many files simultaneously
Local models (Ollama)
Various · Private · Free

Cursor can connect to locally-running models via Ollama. Nothing leaves your machine. Quality is lower than cloud models, but useful for privacy-sensitive code or working offline.

Best for: Code you can't send to external APIs, offline work

Try it out — a 10-minute exercise

Cursor's power is in its four interaction modes. This exercise walks you through all four — so you understand what each one is for and when to switch between them.

Before you start

Open Cursor with any project folder (or create a new empty folder). Create a file called practice.py. In Cursor's settings, choose your model — Claude Sonnet is a good default. Make sure you're on the free trial or have a subscription active.

Step 1 — Tab completion (write a comment, accept the function)

Type this comment and press Enter:

# Return the N most common words in a string

Start typing def. Cursor suggests the entire function. Read what it suggests before pressing Tab. Ask yourself: does this look right? If it's too complex, just keep typing your own simpler version — the suggestion disappears.

Step 2 — ⌘K inline edit (select → describe the change)

Select your entire function. Press Cmd+K (Mac) or Ctrl+K (Windows). A small input appears. Type:

make this case-insensitive and strip punctuation before counting

A diff appears — the original in red, the new version in green. Read every line. Press Cmd+Y to accept or Cmd+N to reject. This is the fastest way to make targeted changes to code you already have.

Step 3 — ⌘L chat (ask a question about your code)

Press Cmd+L to open the chat panel. Type:

Explain what collections.Counter does in this function and why it's the right choice here

The AI explains it in context — it can see your file. Ask follow-up questions as naturally as you would in conversation. This is not Google — you can say "I still don't get the second part" and it will rephrase.

Step 4 — Reference your codebase

In the chat panel, try using the @ reference syntax:

@practice.py Can you add a simple test at the bottom of this file that verifies the function works with a sample sentence?

The AI reads the file you referenced and adds a test that matches your actual function signature. Review it — run it with F5 to confirm it passes.

Step 5 — Control scope deliberately

If Cursor generated more than you wanted — a complex test suite when you wanted one simple test — don't accept it. Press Cmd+N to reject, then re-prompt more specifically:

Just one assert statement, nothing else. Test that "Hello world hello" returns {'hello': 2, 'world': 1}

Specific constraints produce specific outputs. This is a skill — the more precisely you describe what you want, the less you have to trim.

💡
The key habit to build in Cursor

Always read the diff before accepting. Cursor is fast and often right, but "often" is not "always." A 10-second read of the diff catches the mistakes that would cost you 20 minutes to debug later. The goal is speed and quality — you get both by reading, not by accepting everything.

Honest pros & cons

Strengths
✓
Most powerful AI integration available

No other mainstream editor gives AI the same level of codebase access. The @codebase feature and agent mode are qualitatively ahead of what VS Code extensions offer.

✓
Familiar if you know VS Code

The transition cost is nearly zero. Import your settings, and it feels like home with better AI. This is a big deal — most tools require learning a new environment.

✓
Model flexibility

Swap between Claude, GPT-4o, Gemini, and local models. Pick the best tool for each task rather than being locked into one vendor's model.

✓
Agent mode actually works

Multi-file agentic coding in VS Code feels like an add-on. In Cursor it's a first-class workflow, and it shows — the agent mode is more reliable and more capable.

Limitations
✗
Subscription required for serious use

The free tier gives limited AI requests. Real use requires ~$20/month. VS Code with Copilot is similar in cost, but Cursor's free tier is more restricted.

✗
Newer — less battle-tested

VS Code has been around since 2015. Cursor since 2023. There's less community knowledge, fewer tutorials, and the product still changes rapidly. Occasional rough edges.

✗
Agent mode requires careful review

The same power that makes agent mode valuable also makes it risky if you accept changes without reading them. It confidently makes mistakes. The human review step is not optional.

✗
No equivalent to PyCharm's Python depth

Cursor inherits VS Code's Python support (via the Python extension). PyCharm's native Python intelligence — especially the debugger and refactoring tools — is still deeper.
