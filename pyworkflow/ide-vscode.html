What VS Code actually is

VS Code is a source code editor — technically not a full IDE, but so extensible that the line has blurred. Built by Microsoft, it launched in 2015 and has become the most-used development tool in the world. It starts as a lightweight text editor that understands code: syntax highlighting, basic autocomplete, a built-in terminal, file explorer, and Git integration out of the box.

The real power is the extension marketplace. Every capability you'd want — Python language support, database tools, Docker, Jupyter notebooks, AI assistants — is installed as an extension. This means VS Code can become whatever you need it to be. It's the LEGO approach to an IDE: the base is minimal and fast, and you add exactly the blocks you want.

The core editor

The base VS Code gives you: a fast file browser, split panes, an integrated terminal, Git diff views, search across all files, and a command palette (Cmd+Shift+P) that gives you keyboard access to every feature. This alone is more capable than Notepad or TextEdit.

The Python extension

Installing Microsoft's Python extension transforms VS Code into a Python-aware IDE: real-time type checking, jump-to-definition, automatic import suggestions, a debugger, and test runner integration. This extension is what makes VS Code viable for Python work.

The AI layer

One or more AI extensions (Copilot, Claude, Gemini, etc.) add the AI capabilities. These run on top of the base editor and communicate with cloud AI models. Because AI extensions are just extensions, you can mix and match them, or switch between them.

What AI can do inside VS Code

AI in VS Code isn't one thing — it's several distinct capabilities that each solve a different problem. Understanding all of them is the difference between using 10% of what's available and using it well.

1

Inline autocomplete GitHub Copilot · Tabnine · Codeium

As you type, the AI watches and suggests what comes next — sometimes a word, sometimes a whole function. The suggestion appears in gray; press Tab to accept it or keep typing to ignore it. This is the most visible and continuous form of AI assistance — it's on constantly, in the background, inferring intent from your code and comments.

How it works in practice

You write a comment: # calculate compound interest for given principal, rate, and years and press Enter. Copilot generates an entire function body below it. You review it, accept or modify, and move on. The comment was the prompt.

Best for: Repetitive patterns, standard library usage, boilerplate code, anything where "the obvious next thing" is what you want.
2

Chat panel GitHub Copilot Chat · Claude for VS Code · Gemini Code Assist

A conversation interface embedded in the IDE sidebar. You can ask questions, paste code, describe problems, and get detailed responses — without leaving the editor. Unlike browser-based chat, the IDE chat can reference your open files directly using @workspace or #file syntax.

What you can ask
  • "Why is this function throwing a KeyError on line 42?"
  • "Rewrite this loop to be more Pythonic."
  • "What's the difference between append and extend?"
  • "How should I structure this project if it's going to have 10 modules?"
  • "Add type hints to every function in this file."
Best for: Understanding code, getting explanations, asking design questions, debugging problems you can't figure out alone.
3

Inline edits (edit mode) GitHub Copilot · Cursor (built-in)

Select a block of code, press a keyboard shortcut, describe what you want changed, and the AI rewrites it in place. The original and new version appear side-by-side (a "diff view") and you accept or reject the change. This is different from chat — it directly modifies your code rather than suggesting in a panel.

Example

Select a 20-line function → press Cmd+I → type "add error handling for missing keys and log the error" → the AI rewrites the function with try/except blocks. You review the diff and click Accept.

Best for: Targeted improvements to existing code — adding error handling, refactoring, adding logging, improving variable names.
4

Code explanation All AI extensions

Right-click on any code → "Explain this" (or equivalent). The AI reads the selected code and writes a plain-English explanation of what it does, why it's written that way, and any potential issues. This is one of the most underrated uses — it works just as well on code you wrote six months ago as on unfamiliar library code.

Best for: Understanding inherited code, library internals, unfamiliar patterns, or anything you wrote when you were smarter than you currently feel.
5

Test generation GitHub Copilot · Claude

Select a function → ask the AI to generate unit tests for it. It reads the function's logic, identifies edge cases, and writes tests using your existing test framework (pytest, unittest). This dramatically reduces the cost of writing tests, which most developers skip because it's tedious.

Example prompt

"Write comprehensive pytest tests for this function. Cover the happy path, empty inputs, invalid types, and any edge cases you can identify."

Best for: Building out test coverage quickly, catching edge cases you hadn't thought of, learning what to test.
6

Documentation generation GitHub Copilot · Claude

Select a function or class → ask for a docstring. The AI reads the code and writes documentation: what it does, what the parameters are and their types, what it returns, and what errors it might raise. It can also generate README files for your whole project.

Best for: Adding documentation to existing code, maintaining a consistent doc style, anything you're going to share with others.
7

Agentic / multi-file editing GitHub Copilot Workspace · Claude Code extension

Give the AI a higher-level task — "add a user authentication system" or "refactor all database calls to use this new ORM" — and it plans what changes to make across multiple files, shows you a preview, and executes with your approval. This is the most powerful and the newest capability; it's still maturing but already handles many real tasks.

Best for: Large refactors, adding a new feature that touches many files, repetitive changes across a codebase.
8

Error diagnosis GitHub Copilot · Claude

When your code throws an error, the AI can be invoked directly on the traceback. It reads the error message, traces it back to the source, and explains what went wrong and how to fix it. Some integrations can do this automatically when an error appears in the terminal.

Best for: Error messages you don't understand, dependency conflicts, environment issues, anything where the traceback is confusing.

What a real session looks like

This is a moment-by-moment description of what a developer with AI assistance actually does. Not theory — this is the actual sequence of events in a typical work session.

Opening

Open folder → environment activates

You open your project folder in VS Code. The Python extension detects your .venv and activates it. The status bar at the bottom shows which interpreter is selected. Copilot loads silently in the background.

First few minutes

Write a comment → Copilot suggests the code

You start writing. Type a comment describing what you want to do. Copilot reads it and suggests the implementation below. You review the suggestion — sometimes it's exactly right, sometimes it needs adjustment. You accept with Tab or modify manually.

# Load CSV data and remove rows where price is null
↓ Copilot suggests:
df = pd.read_csv('data.csv')
df = df.dropna(subset=['price'])
df['price'] = df['price'].astype(float)
When stuck

Open chat → ask the question

You hit something you don't know — how to merge two dataframes on a shared column. Press Ctrl+Shift+I to open Copilot Chat. Ask in plain English. It answers with code and explanation. You paste the relevant part into your file.

Error occurs

Run → error → explain

You run the file. An error appears in the terminal. Right-click on the error message → "Explain this error." The AI reads the traceback and tells you exactly what went wrong and where to look. Usually faster than reading the traceback yourself.

Code review

Select code → ask for improvement

Your code works, but you suspect it could be cleaner. Select the function, open chat: "Is there a better way to write this? I'm concerned about performance with large datasets." The AI suggests a vectorized approach, explains why it's faster, and offers the rewrite. You decide whether to take it.

End of session

Commit → Copilot suggests commit message

You open the Source Control panel, stage your changes, and click the commit message field. Copilot reads your diff and suggests a descriptive commit message. You review, edit if needed, and commit.

💡
The mental shift

Developers who use AI well don't just ask "write this for me." They treat it as a conversation — they share context, they ask follow-up questions, they push back on suggestions they disagree with. The quality of your AI output scales directly with the quality of context you provide. Vague prompt → vague code. Specific prompt with constraints → specific, useful code.

Setting up AI in VS Code

The AI extensions you install depend on your goals. Here are the main options and how to get each one running.

GitHub Copilot
~$10/mo · Free for students

The most integrated option. Extensions: "GitHub Copilot" (inline suggestions) + "GitHub Copilot Chat" (conversation panel). Both install from the Extensions marketplace. Sign in with your GitHub account. Copilot begins working immediately — gray suggestions as you type, chat panel available in the sidebar.

Extensions → search "GitHub Copilot" → Install
Install "GitHub Copilot Chat" as well
Sign in with GitHub when prompted
Start typing — gray suggestions appear automatically
Claude (via Anthropic extension)
Free tier + API key required

Search for "Claude" in the Extensions marketplace. The official extension (or well-reviewed community extensions) give you a chat panel backed by Claude. You'll need an Anthropic API key from console.anthropic.com. Claude is especially good for complex explanations and code reasoning.

Get API key at console.anthropic.com
Extensions → search "Claude" → Install
Add API key in extension settings
Open chat panel and start asking questions
Gemini Code Assist
Free (Google account required)

Google's AI coding assistant for VS Code. Search "Gemini Code Assist" in Extensions. Sign in with a Google account. Provides inline suggestions and a chat panel. A genuinely free option for those who don't want to pay for Copilot.

Extensions → search "Gemini Code Assist" → Install
Sign in with Google account
Inline suggestions and chat are immediately available
Codeium
Free forever (with limitations)

A well-regarded free alternative to Copilot. Provides inline autocomplete and chat. Supports VS Code, PyCharm, and JetBrains products. Good choice if you want AI coding help without any subscription cost.

Extensions → search "Codeium" → Install
Create free account at codeium.com
Sign in within VS Code
💡
You can run multiple AI extensions simultaneously

Many developers run Copilot for inline suggestions and Claude (via a separate extension or browser tab) for more complex conversations. They serve different purposes and don't conflict with each other.

Honest pros & cons

Strengths
✓
The AI ecosystem gravitates here first

Every new AI coding tool releases a VS Code extension first. Copilot, Claude, Gemini, Codeium, Continue — all are available. You'll never be blocked from using a new AI tool because your editor doesn't support it.

✓
Free and open source

The editor itself costs nothing. The AI tools have varying costs, but the editor — including all its core capabilities — is free forever.

✓
Enormous community

Whatever problem you hit, someone else has hit it first. Stack Overflow, YouTube, Reddit — VS Code questions have answers everywhere. This is especially valuable when you're learning.

✓
Works for every language

If you ever move beyond Python — to JavaScript, R, SQL, TypeScript, anything — VS Code handles it. You're learning one editor for your entire career.

✓
Lightweight and fast

Opens instantly. Doesn't consume significant memory when idle. This matters more than it sounds when you have many other things running.

Limitations
✗
Python support requires configuration

Out of the box, VS Code is not Python-aware. You need to install and configure the Python extension, select the right interpreter, and set up your virtual environment. PyCharm handles all of this automatically.

✗
Extension conflicts and instability

With many extensions installed, things occasionally break. An update to one extension can cause problems with another. PyCharm's integrated approach avoids this entirely.

✗
Debugger is good but not the best

VS Code's Python debugger is solid. PyCharm's is exceptional. If you do a lot of step-through debugging of complex code, PyCharm has the edge.

✗
No built-in virtual environment management UI

Creating and managing virtual environments is done via the terminal or command palette. PyCharm has a visual UI for this. Small difference, but relevant for beginners.

Try it out — a 10-minute exercise

This walks you through the complete AI-assisted coding loop in VS Code, from writing a function to running it and refining it with AI help. By the end you'll have used every major AI capability at least once.

Before you start

Make sure you have VS Code open with the Python extension installed, a virtual environment active, and GitHub Copilot (or any AI extension) installed. Create a new file called practice.py.

Step 1 — Let AI write the first function

In practice.py, type this comment on the first line and press Enter:

# Calculate the percentage change between two numbers

Copilot will suggest the function. Press Tab to accept it. Read what it wrote — don't just accept blindly. Does it handle the case where old_value is zero? If not, that's something to fix.

Step 2 — Run it and see what happens

Add two lines below the function and run the file:

result = percentage_change(100, 150)
print(result)

Press F5 (or right-click → Run Python File). You should see 50.0 in the terminal. If you get an error, that's fine — read it carefully. It usually tells you exactly what's wrong.

Step 3 — Ask AI to explain what it wrote

Select the entire function with your mouse. Then open the Copilot Chat panel (the speech bubble icon in the left sidebar, or Ctrl+Shift+I). Type:

Explain this function line by line. What does it return and why?

Read the explanation. This is the learning loop — AI writes, you ask it to teach you. You should understand every line before moving on.

Step 4 — Ask AI to improve it

In the chat panel, type:

Rewrite this function to handle the case where old_value is zero. Return None in that case and print a warning message.

The AI will rewrite it. Read the new version. Does it look right? If you don't understand a part, ask: "What does the if statement on line 3 do?"

Step 5 — Control how much AI generates

If the AI wrote too much or too little, you can always guide it. Try:

  • "Keep it shorter — just the function, no extra comments"
  • "Add a docstring explaining the parameters"
  • "Show me a simpler version with no error handling"

You're in charge of scope. AI responds to constraints. The more specific you are, the more targeted the output.

Step 6 — Run the final version

Test the improved function with a few inputs:

print(percentage_change(100, 150)) # should print 50.0
print(percentage_change(0, 100)) # should handle zero
print(percentage_change(200, 100)) # negative change

Run it. Verify the output makes sense. You've now done the full loop: comment → AI writes → run → ask AI to explain → improve → run again. That's the core workflow.

💡
The key takeaway from this exercise

AI is most useful when you stay in control of the scope. One focused request gets you one useful function. "Write me a complete data analysis tool" gets you something overwhelming. Start small, read everything, refine with follow-up prompts, and run after each change. That rhythm — small request, read, refine, run — is the actual workflow.

Who VS Code is for

Strong fit if you...
  • Work in multiple languages or plan to
  • Want to use multiple AI tools and compare them
  • Value a large community and lots of tutorials
  • Want the editor most commonly used in industry
  • Prefer to configure your own workflow
  • Are cost-conscious (the editor is free)
Consider PyCharm instead if you...
  • Work exclusively in Python and want zero configuration
  • Do heavy debugging with complex codebases
  • Want a UI for managing packages and environments
  • Are willing to pay for a more polished Python experience
